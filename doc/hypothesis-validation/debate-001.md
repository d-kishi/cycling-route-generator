## Round 1 各ペルソナのアイデア(集計)

### persona-hobby (中級ホビーロードサイクリスト) — 4個提案

1. **未踏率ベースの新規ルート提案エンジン**: 自分のStrava走行履歴から未踏率の高いエリアを優先し、「過去2年で1度も走っていない道を60%以上含むルート」を提案
   - 出典: BikeForums(2019)mcours2006氏 / PistonHeads UK(2024)Correvor氏
   - 対立: 初心者層と「安全性軸」、競技志向層と「目的関数」で対立
2. **機材沼トリガー連動の試走ルート提案**: 新パーツ登録で試走に最適なルートを自動提案(軽量ホイール=登り、エアロ=平坦巡航、グラベル=ダート)
   - 根拠: motivation「新パーツ・機材を試す楽しみ」、annual_maintenance_cost 30万円/年
   - 対立: カジュアル層と「テーマの粒度」、初心者と「機材を持たない」
3. **行きつけショップ起点ルート + 立ち寄りスポット**: ホームショップ発着の周回ルート、自転車関連スポット表示で機材トラブル時の安心とカルチャー文脈の寄り道を両立
   - 根拠: maintenance_practice「特定ショップとの継続関係」
   - 対立: 初心者は「ホームショップ前提が成立しない」、ロング志向は「片道計画と相反」
4. **Stravaセグメント密度ヒートマップの逆利用**: セグメント密度中庸のエリア(=他者の知見はあるがメジャーすぎない)を優先しルート生成
   - 出典: businessofapps.com Strava有料会員2%
   - 対立: 競技志向と「セグメントの使い方」(KOM狙い vs 中庸)

### persona-beginner (ビギナーロードサイクリスト) — 4個提案

1. **「初心者卒業ロードマップ」型ルート段階処方**: 距離・標高・車道率・信号密度4軸の初心者難易度スコアで「次に挑戦すべきルート」を1-3本だけ提示。**処方箋型UI**
   - 出典: sbaa-bicycle.com「乗る→乗りこなす」/ milbon24.com 30→50→100km
   - 対立: 中級・競技と「お節介機能」「広げたい vs 絞ってほしい」が真逆
2. **車道率・信号密度ヒートマップ + 安心ルート優先モード**: 車道率%・信号密度・歩道率・事故報告地点を表示しデフォルトで車道率低い順ソート、飛び出し多発・小学生登下校路の時間帯警告
   - 出典: Yahoo!知恵袋「路肩なし車道恐怖」/ bicycle-post.jp「飛び出し」
   - 対立: ロング志向と「効率性 vs 安全性」、競技は「信号密度低い=練習向き」と意味が逆
3. **途中撤退・輪行リカバリー前提のルート生成**: 5km毎の最寄り駅・コンビニ・自転車店マーカー、「パンク発生想定」で輪行可能駅まで即時表示
   - 出典: ryusuke-tax.com「1人で走行は怖い、地名を知らないと迷う」
   - 対立: 競技・ロング志向と「ノイズ情報」(駅マーカー不要)
4. **初心者向けお手本ルート図鑑(コミュニティ非依存)**: 運営/ショップ協力で事前検証された地域別「初心者用お手本ルート」プリセット集
   - 出典: sbaa-bicycle.com「30km以内・CR・河川敷」/ akira779.com グループライド参加心理障壁
   - 対立: カジュアル層と「テーマ性 vs 安全性」、中級層と「運営手作業のスケール問題」

### persona-serious (シリアスロードライダー) — 4個提案

1. **ワークアウト連動ルート生成 (TrainingPeaks/Zwift構造化ワークアウト→屋外路上投影)**: VO2max 4分×5本等のメニューに地形が一致する屋外ルート(勾配4-6%・距離1.5-2km・信号ゼロ)を自動生成、Garmin Edge にコース+ワークアウト同時転送
   - 出典: note.com FTP頭打ち / DCRainmaker TrainingPeaks構造的分離批判
   - 対立: カジュアル・初心者と「テーマ性 vs パワー出力」、ロング志向と「補給最適化」
2. **ヒルクライム試走モード (本番×当日コンディション再現)**: 富士ヒル/乗鞍/ツール・ド・おきなわ等の主要レースコースをカタログ化、3週前/1週前/6週前で期分け試走切替、本番コース近隣で標高プロファイル類似の代替試走ルート提案
   - 出典: note.com/commutebymybike 富士ヒル試走「平地260W→高地230W」
   - 対立: 中級層と「飽き vs 本番再現性」(価値観が真逆)
3. **チーム練/集団走行ルート専用モード**: 道幅5m以上・信号間隔3km以上・直線1km以上をフィルタ、人数とローテ時間で生成、修善寺/群馬CSC等の周回プリセット、先頭交代タイミングGPXウェイポイント挿入
   - 出典: チーム練習固有要件(broadlistening)
   - 対立: 初心者と「安全性の定義」(裏道vs幅広直線)、カジュアル層と「速度域」(35-40km/h vs 20km/h)
4. **オーバートレーニング兆候連動ルート強度ダウンシフト**: HRV・安静時心拍・睡眠スコア+CTL/ATL/TSBで当日フォーム判定、TSB<-30+HRV低下7日連続なら強度半減ルートに自動置換
   - 出典: zwiftworker 9原因 / Red Bull / 栗村修コラム
   - 対立: 初心者・カジュアルと「データ駆動 vs 気分駆動」(指標自体を持たない)、ロング志向と「サイクル単位」が異なる

### persona-casual (カジュアルスポーツバイク乗り) — 4個提案

1. **テーマ起点ルート生成(目的地ではなく"巡り方"から始まる入力フォーム)**: テーマプリセット(御朱印/カフェ/酒蔵/聖地巡礼/お花見/廃線跡)+所要時間+寄り道数で入力、AIが寄りどころ3-5個ピックアップして連結
   - 出典: つくば霞ヶ浦りんりんロード「ツール・ド・御朱印」/ Toy Bike お花見ライド
   - 対立: 競技・ロング志向と「主入力の優先度」(数値 vs テーマ)
2. **駐輪可否マップ + 自転車で行きづらい店フラグ**: 各経由スポットに駐輪場有無/路上可・公共交通アクセス難易度/営業時間・定休日をルート上に表示
   - 出典: wimo.co.jp カフェ巡り実践障壁 / 知恵袋「ストリートビュー確認」
   - 対立: ロング・競技と「ルート上の情報密度」(駐輪はノイズ)
3. **旅先モード(出発点が自宅じゃない・装備借り物・知識ゼロ前提)**: レンタサイクル店舗住所起点、滞在時間と借りる自転車グレードで勾配・距離自動調整、地元の人しか知らない路地の有無設定
   - 出典: TraveVoice しまなみ2025年度7万9502件・インバウンド70%増 / TABIRIN 中間距離需要
   - 対立: 中級・ロング志向と「ルートの再利用性」(地元蓄積 vs 旅先使い捨て)
4. **Stravaを持っていない人のためのエクスポート(GPX/TCX以外の出口)**: Googleマップで開けるリンク(複数経由地URL)/印刷向けスクショ/LINE共有リンク
   - 出典: Yahoo!知恵袋「NAVITIME無料は検索のみ」
   - 対立: 競技/ロング/中級と「真っ向から対立」、GPXエクスポート最重要vsカジュアル代替UI

### persona-endurance (ロングライド/ブルベライダー) — 4個提案

1. **PC兼補給ポイント自動配置ルート生成**: コンビニ空白地帯(連続30km以上)ハイライト、道の駅営業時間・自販機分布の重畳、グロス速度14km/h前提のハンガーノック警戒区間マーキング、通過予想時刻の営業中/閉店中判定
   - 出典: ikarugasan「25kmごとコンビニマーク手作業」/ 国交省道の駅要綱(地域振興施設は24h義務なし) / JSNA 30-60g/h補給推奨
   - 対立: カジュアルと「ノイズが多すぎる」、競技志向と「補給より練習効率(空腹耐性訓練)」
2. **グロス速度シミュレータ(仮眠・補給込みで完走可否事前判定)**: ルート+巡航速度+補給滞在パターン+仮眠戦略を入力 → 各PC通過予想時刻・制限時間に対する余裕・グロス速度14km/h以下警告・仮眠取得最大値
   - 出典: potaride DNF分析 / Audax Japan BRM制限時間 / note.com/anna241e PBP2023体験記
   - 対立: 初心者と「数字が威圧的」、カジュアルと「ライドは時間管理じゃない」
3. **GPSデバイス充電前提のルート設計**: サイコン機種別(Edge 530/830/1040等)の公称稼働時間・劣化率反映、バッテリー残量シミュ、充電ポイント候補(イートインありコンビニ・コンセント開放道の駅)地図重畳、ハブダイナモ発電シミュ、外部バッテリー脱落リスク考慮
   - 出典: BikeForumsランドヌール「内蔵電池400km限界」/ cannonball24「Edge 530の2.5年で6割」
   - 対立: 初心者・カジュアルと「機材オタク向けすぎる」、競技と「ハブダイナモ重量増許容」
4. **紙キューシートPDF出力(RWGPS Premiumなしで二重保険)**: ブルベ向け書式(交差点距離・方向・PC・累積距離・累積標高・通過予想時刻入)PDF、PCと交差点キューを分離出力、A4二つ折り防水紙対応、雨天劣化想定の縮小/拡大/PC情報のみ版同時出力
   - 出典: potaride「8割が紙キューシート」「雨で15km迷った」/ BikeForums「RWGPS交差点キュー欠落」/ RWGPS仕様 Cue Review Premium限定
   - 対立: 競技と「紙文化は時代遅れ」、カジュアルと「機能過多」、中級と「優先度低い」

## Round 2 各ペルソナの応答(要点)

各ペルソナがD1-D8それぞれに反応。SendMessageでRound 1 agentに継続させ、agentIdベースで送信(persona-yaml再送不要)。

### persona-hobby Round 2 要点

- **D1**: 「安全担保の上での未踏」(OSM二輪通行可・幹線国道率20%以下デフォルト)+UIで「目的」モード分離
- **D2**: 数値+軽量テーマ語彙のハイブリッド、階層UI(プリセット/テーマピル/詳細パネル)
- **D3**: GPX/.tcx は死守、比較フェーズでGoogleマップリンクは層横断機能
- **D4**: マーカー類は全レイヤー化してON/OFF
- **D5**: hobby層には重すぎる、データ駆動は上位プラン分離を強く要求(D7と連動)
- **D6**: 前提でなくオプション、「任意の起点登録」抽象化で全層対応
- **D7**: MVPはhobby最適化、サブスクより「機能モジュール段階解放」、BRouterセルフホスト活かして無料・広告なしを死守
- **D8**: 「層プリセット+テーマピル+詳細展開」の三層UI、テーマ語彙は層横断ライブラリ

### persona-beginner Round 2 要点

- **D1**: 「初心者向けに事前検証された定番」優先、経験レベル別に未踏率レンジ反転(初心者20%以下/中級60%以上)、「経験・目的に応じて新規性を自動調整」が3層を束ねる上位概念
- **D2**: 処方箋型UIをデフォルト、テーマ派と処方箋派は併存可能、「パラメータを入れない、選ぶだけ」のUIモードで層別中身を変える
- **D3**: 三系統提案(競技GPX/初心者簡易PDF/カジュアル共有リンク)、Strava前提は初心者の足切り
- **D4**: 「同じデータ、表示は層プロファイルで制御」、初心者デフォルトON、ベテランOFF
- **D5**: 関係薄いが「気分駆動」は初心者にも適用可、「今日は元気/疲れ気味」3択UIで serious 機能を初心者向け簡素化派生
- **D6**: 障壁、逆に「初めてのショップ訪問ルート」「初心者歓迎フラグ付きショップ経由ルート」が有用
- **D7**: 強く是正要求、「最初の1ライド」体験を初心者基準で設計、後の機能は層別分岐
- **D8**: 「目的選択トップ(健康維持/景色/トレーニング/ロング/テーマ)→層別分岐」の2段階UI

### persona-serious Round 2 要点

- **D1**: 期分け時間軸分離(Base=未踏率高、Build=地形マッチング、Race直前=本番再現、Recovery=未踏率高)。「未踏率を季節・期分けプロファイルでウェイト調整可能」スライダーで全層統合
- **D2**: 数値+構造化ワークアウト言語(.zwo/.erg)入力、プロファイル切替で UI 完全差し替え
- **D3**: 二系統完全並列、OAuth連携でワンクリックpush必須(Garmin/TrainingPeaks/Strava)。endurance紙PDFはテーパリング期に使える層横断機能
- **D4**: トレーニング目的別の表示プロファイル切替、インターバル中は全OFF、Z2 Base 200km走時はON
- **D5**: コア機能、上位プラン限定で隔離、hobby有料勢にはCTLのみ部分提供の余地
- **D6**: 全プロファイル共通実装で良い(意味は層ごとに違う)、機材依存度高で特定ショップ集中依頼
- **D7**: 具体的サブスク提案 → **Free / Pro $10-15 / Race $25-30 / チーム年契約**、MVPはhobby単独立ち上げ→ARR一定後にPro追加の時間軸分離
- **D8**: 完全分離、初回5問アンケート自動判定、共通エンジン+プロファイル別フロントエンド3-4種

### persona-casual Round 2 要点

- **D1**: 関係薄い、「未踏率」より「テーマ充足度」「未テーマ率」が指標(同エリアでもテーマが変われば新規体験)
- **D2**: テーマ派+処方箋派は連合可能、デフォルトを数値にすると初心者・カジュアル離脱
- **D3**: 三系統必要(GPX/Googleマップ/印刷PDF+LINE共有)、紙PDFと印刷コース図は連合、出口の並び順すら層別
- **D4**: 「補給」「駅マーカー」はノイズ寄りだが「駐輪可否/カフェ営業時間/観光スポット定休日」が必須、マーカー種別を層別ON/OFFで対立解消

### D1: 未踏率最大化 vs 安全な定番 vs 本番再現性

| 層 | 主張 |

| endurance | 「累積標高プロファイル類似度」「PC間隔再現性」 |

**統合提案**: 「ルートの新規性軸」を**ユーザの経験・目的・期分けで自動調整するスライダー**として実装。「未踏率」(GPSログ前提)と「未テーマ率」(テーマ非依存)は異なる軸として並列実装。

**残存対立**: 「階層UI内で全層吸収」(hobby) vs 「完全分離」(serious/casual/endurance)→ 完全分離が多数派
